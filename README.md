# Django local test

Notes and reproduction code on triage, review, etc. of [Django](https://github.com/django) tickets.

## What is this for?

For me to keep notes and share reproduction code on Django tickets and PRs that I triage and review.

## Branches
* **main** - Notes for all tickets reside in the `notes/` directory.
* **boilerplate** - Bare minimum Django project starter code for local development.
* **ticket-*** - Each ticket's bug reproduction code.

Notes are not polished.

Boilerplate for testing against Django core inspired by [@MHLut](https://github.com/MHLut)'s wonderful blog post, [A Simple Approach To Running Django Core Locally](https://marijkeluttekes.dev/blog/articles/2024/01/25/a-simple-approach-to-running-django-core-locally/), and repository, [django-local-test](https://github.com/MHLut/django-local-test).
